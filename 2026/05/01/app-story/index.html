title:从小程序到安全驱动的AP应用开发复盘

date: 2025-04-01 17:55:00

categories: 项目复盘

tags: 网络安全,Java,Python,项目开发,比赛心得

-–

最近，我参加了一场开发类比赛。

最开始，我满怀期待、斗志满满，打算从零开始做一款小程序。可真正动手后才发现，事情远没有想象中顺利——需求不断变化、思路逐渐偏离最初的想法，越做越失落，甚至一度怀疑自己能不能完成作品。

更棘手的是，开发过程中遇到了各种平台限制，很多我想实现的功能根本无法落地。眼看着比赛即将结束，我在最后几天做了一个重要决定：**放弃小程序，重新转向AP应用开发**。

在重构项目的过程中，我突然想通一件事：

任何产品，都必须突出核心主题；而我平时对网络安全特别感兴趣，那我的作品，为什么不能把「安全」当成核心亮点？

于是，我开始接触学习网络安全相关知识，也真正意识到：

**一个让用户放心的产品，安全永远是底线。**

-–

## 一、开发中的安全思考：为什么我从 Python 转向 Java？

最开始，我用的是 Python 进行开发。

Python 上手快、写爬虫方便、效率高，我也确实用它完成了数据爬取、接口调用等功能，学到了很多实用技巧。

但随着深入了解安全知识，我发现一个关键问题：

**Python 的语言安全性相对较弱。**

简单说：

- Python 是解释型语言，代码容易被反编译

- 内存管理、权限控制、安全沙箱机制不如静态语言严谨

- 做后端服务时，面对攻击的防护能力有限

而我的项目需要对外提供服务，还要接入域名、服务器、AI API，涉及用户数据与接口调用，**安全绝对不能妥协**。

所以，我最终决定：

**改用 Java 编写后端。**

Java 的优势非常明显：

- 强类型语言，安全性更高

- 生态成熟，安全框架丰富

- 适合做稳定、可靠的后端服务

- 企业级开发主流，更符合工程规范

这次技术选型的转变，让我第一次真正理解：

**开发不只是实现功能，更是对安全、稳定、用户信任的负责。**

-–

## 二、网络安全到底有多重要？我用实战明白了

在这次开发中，我接触了很多以前不了解的安全知识点，也真正意识到网络安全离我们有多近：

1. **API 安全**

我接入了 AI 接口，才知道 API 如果不加验证、不加签名，很容易被恶意调用，造成费用损失、数据泄露。

2. **数据传输安全**

用户数据在网络上传输时，如果不加密，就像明信片一样，任何人都能看到。

所以必须使用 HTTPS、加密算法。

3. **后端权限控制**

没有权限校验的接口，任何人都能随意篡改、删除数据。

这就是为什么企业项目必须做登录验证、身份认证、权限隔离。

4. **爬虫与反爬**

我用 Python 写过爬虫，也明白爬虫如果滥用，会对别人服务器造成伤害；

反过来，自己的服务器也要做好反爬防护。

这些知识，以前只是听说，这次真正用在项目里，才彻底懂了。

-–

## 三、项目落地：域名、服务器、AI 接入，每一步都是成长

确定方向和技术栈后，我开始真正把项目跑起来：

- 申请域名

- 配置服务器

- 部署后端服务

- 接入 AI API

- 做接口调试

- 加固安全策略

每一步都花了大量时间，也踩了无数坑。

但正是这些琐碎、枯燥、反复调试的过程，让我从一个只会写点小代码的新手，慢慢变成能独立完成**完整项目上线**的开发者。

-–

## 四、总结：这场比赛，我收获的不只是一个项目

回头看这次竞赛经历：

- 从小程序到 AP 应用，我学会了**快速调整、灵活重构**

- 从只会写功能，到重视安全，我建立了**工程化思维**

- 从 Python 到 Java，我理解了**技术选型的真正意义**

- 从零基础搭建服务器、域名、API，我完成了**全栈开发的第一次实战**

最重要的是，我真正理解了一句话：

**网络安全不是附加题，而是所有产品的必修课。**

未来，我会继续深耕网络安全与后端开发，努力做出更安全、更可靠、更让用户放心的产品。

这不仅是一次比赛，更是我技术道路上的重要起点。
